# atmina — Political Transparency Platform Design Spec

**Date:** 2026-04-06
**Author:** atmina.lv
**Status:** Approved

---

## 1. Product Overview

**atmina.lv** is a public political transparency platform for Latvia. It tracks what politicians say, how they vote, and where words and actions diverge. All data is structured, source-linked, searchable, free, and ad-free.

**agents.atmina.lv** is a landing page for the paid service — custom political intelligence agents built for parties, journalists, and organizations. For launch, it's a simple page with capabilities overview and CTA. Evolves into an interactive demo later.

**Core proposition:** Political memory. News portals publish articles that disappear into archives. atmina.lv makes political data permanent, structured, and explorable. Every position, every vote, every contradiction — with source links, forever.

## 2. Architecture

### 2.1 High-Level

```
Scraping Pipeline          Analysis (Claude Code)       Static Site Generator
┌──────────────┐          ┌──────────────────┐          ┌─────────────────┐
│ 9 news sites │──┐       │ Claim extraction  │          │ generate.py     │
│ X/Twitter    │──┤──DB──→│ Contradiction det │──DB+Wiki→│ Jinja2 templates│──→ output/
│ Saeima votes │──┘       │ Neutral briefs    │          │ Chart.js        │
└──────────────┘          │ Wiki updates      │          └─────────────────┘
                          └──────────────────┘
```

- **Database:** SQLite + sqlite-vec (384-dim embeddings). Single file: `data/atmina.db`
- **Wiki:** Obsidian vault (`wiki/`). Frontmatter auto-synced from DB, body is manual editorial content.
- **Output:** Static HTML site generated by `src/generate.py`. Deployed to any static hosting.
- **Routine:** Daily, interactive, driven by user + Claude Code. No automation for launch.

### 2.3 Tech Stack

- **Language:** Python 3.11+
- **Database:** SQLite (WAL mode) + sqlite-vec extension (384-dim embeddings via `intfloat/multilingual-e5-small`)
- **Validation:** Pydantic v2
- **Templating:** Jinja2
- **Scraping:** httpx + trafilatura + BeautifulSoup4 + twikit (X/Twitter cookie-based auth)
- **NLP:** simplemma (Latvian lemmatization), fasttext (language detection)
- **Charts:** Chart.js (bundled, no CDN)
- **Blog rendering:** markdown (Python package, with tables + fenced_code extensions)
- **Frontend:** Vanilla CSS + JS. No framework. Dark theme. Responsive.

## 4. Wiki (Obsidian Vault)

### 4.1 Design Principles

- The wiki stores what the DB cannot: understanding, context, analysis, narrative.
- Frontmatter (YAML) is auto-synced from DB — stats only. Body is manual, never overwritten.
- No `<!-- PROTECTED -->` markers. Separation is structural: YAML = auto, body = manual.
- Every politician and topic gets a stub page (frontmatter only) on first sync. Editorial content grows organically.
- Wiki is dual-purpose: Claude Code context + content source for atmina.lv public pages.

### 4.2 Page Format

**Person page (`wiki/persons/Evika Siliņa.md`):**
```markdown
---
name: Evika Siliņa
party: Jaunā Vienotība
role: Ministru prezidente
claims: 45
contradictions: 3
votes: 18
last_active: 2026-04-05
topics: [Sociālā politika, Koalīcija un partijas, Budžets un finanses]
---

(Manual editorial content below — never touched by wiki_sync)
```

**Topic page (`wiki/topics/Aizsardzība un drošība.md`):**
```markdown
---
topic: Aizsardzība un drošība
claims: 134
politicians: 28
contradictions: 5
last_activity: 2026-04-05
top_politicians: [Jānis Dombrava, Andris Sprūds, Raimonds Bergmanis]
---

(Manual editorial content below)
```

**Synthesis page (`wiki/synthesis/slesers-nato.md`):**
```markdown
---
title: Šlesera NATO evolūcija
politicians: [Ainārs Šlesers]
topics: [Aizsardzība un drošība, Ārpolitika]
created: 2026-04-06
---

(Fully manual analysis content)
```

**Daily page (`wiki/dailies/2026-04-06.md`):**
```markdown
---
date: 2026-04-06
documents_ingested: 45
claims_extracted: 12
contradictions_found: 1
---

(Session notes, observations, trends spotted)
```

### 4.3 wiki_sync() Behavior

- Creates stub pages for all active politicians and all 26 topics if they don't exist
- Updates ONLY the YAML frontmatter on existing pages (counts, dates, top lists)
- Never reads, modifies, or deletes body content below the closing `---`
- Updates `wiki/index.md` with full index grouped by party (not relationship_type)
- Appends to `wiki/log.md`

## 5. Static Site (atmina.lv)

### 5.1 Design System

- **Theme:** Dark mode (same aesthetic as politracker dashboard — proven, clean)
- **Colors:** `--bg: #0f1117`, `--surface: #1a1d27`, `--accent: #3b82f6` (blue, not indigo — neutral, not party-colored)
- **Typography:** System fonts (-apple-system, BlinkMacSystemFont, Segoe UI)
- **Layout:** Max-width 1200px, responsive grid, sticky nav
- **Charts:** Chart.js with dark theme defaults
- **No ads. No tracking scripts. No cookie banners.**

### 5.2 Pages

#### Index (`index.html`)
- Hero: "Politiskā atmiņa" + subtitle + election countdown (2026-10-03)
- Stats row: politicians, positions, contradictions, votes
- Latest contradictions (top 6 cards)
- Recent Saeima votes (table, last 10)
- CTA banner linking to agents.atmina.lv

#### Pretrunas (`pretrunas.html`)
- All contradictions as filterable cards
- Filters: party (button bar), severity (button bar), topic (dropdown or buttons)
- Each card: politician name + party, severity badge, topic tag, summary, old stance vs new stance with dates, source links
- Color-coded left border by severity (red = direct, orange = reversal, yellow = shift)
- Client-side JS filtering on embedded JSON data

#### Pozīcijas (`pozicijas.html`)
- Sortable table: politician, party, topic, stance, date, source link
- Filters: party (button bar), topic (button bar), date range
- Client-side JS filtering and sorting

#### Balsojumi (`balsojumi.html`)
- List of votes as expandable cards
- Each card: motif, date, result, par/pret/atturas badges
- Tracked politicians' individual votes shown inline
- Expandable faction breakdown table
- Filter by topic

#### Tendences (`tendences.html`)
- Chart: positions by topic (last 30 days, horizontal bar)
- Chart: most active politicians (last 30 days, horizontal bar)
- Chart: activity timeline (claims + documents, last 30 days, line)
- Time range selector: 7d / 30d / 90d
- Context notes (neutral events/observations) as cards below charts

#### Analīzes (`analizes.html`)
- Hub page linking to deep analysis pieces
- Each analysis: title, date, description, topic tags
- Analyses sourced from `content/analizes/` markdown files with YAML frontmatter
- First analysis: party financing (deklarācijas)

#### Deklarācijas (`deklaracijas.html`)
- Full KNAB party financing analysis
- Stats row: 3,179 donations, 47 parties, EUR 2.35M
- Key findings as highlight cards (JV EUR 760, AS 16.6x ratio, Dandzbergu 53%)
- Interactive charts: top parties bar chart, monthly trends line chart
- Sortable tables: top donors, multi-party donors, spending breakdown

#### Dienas analīze / Blog (`blog.html`)
- Reverse-chronological list of daily/weekly briefs
- Each entry: date, type badge (dienas/nedēļas), title, preview text
- Links to individual post pages

#### Blog Post (`blog/2026-04-06.html`)
- Individual page with permanent URL
- Neutral daily/weekly brief rendered from markdown
- Date, type badge, full content
- OpenGraph meta tags for social sharing
- "Back to all analyses" link

#### Politician Page (`politiki/{slug}.html`)
- Name, party, role
- Stats: positions count, contradictions count, votes count
- Editorial profile from wiki (if written — body content from person page)
- Contradictions section (cards, same format as pretrunas page)
- Positions table (filterable by topic)
- Saeima votes table
- All data with source links

#### Par mums (`about.html`)
- What atmina.lv is
- Methodology: how data is collected (9 sources, X, Saeima), how AI identifies positions and contradictions, confidence scores, 26 topic taxonomy
- Ideology: rendered from `content/ideology.md`
- Disclosure: "AI-assisted analysis. Source links provided for verification. Coverage is selective based on available sources."
- Team: neatkarīgs, anonīms projekts
- Link to agents.atmina.lv for paid services

### 5.3 Data Embedding

All page data is embedded as JSON in the generated HTML. Filtering is client-side JavaScript — no backend, no API calls, instant response. Data is fresh as of last `generate.py` run.

### 5.4 SEO & Sharing

- Every page has `<title>`, `<meta description>`, OpenGraph tags (`og:title`, `og:description`, `og:type`, `og:locale=lv_LV`)
- Politician pages have unique titles: "Evika Siliņa — atmina.lv"
- Blog posts have unique titles and descriptions
- Contradiction cards are designed to look good when screenshotted and shared
- `sitemap.xml` generated alongside the site (post-MVP)

## 7. Neutral Brief Generator

### 7.1 Purpose

Generate daily and weekly political summaries for the atmina.lv blog. No campaign framing, no attack angles, no recommendations — just factual analysis of what happened.

### 7.2 Daily Brief Format

```markdown
# Dienas analīze — 2026-04-06

## Galvenais
- **N dokumenti** (M web + K Twitter/X), **N jaunas pozīcijas** no N politiķiem, **N pretrunas**
- [3-4 bullet points with key developments]

## Aktīvākie politiķi
| Politiķis | Partija | Pozīcijas | Galvenās tēmas |
[Top 10 by claim count]

## Galvenās tēmas
### 1. [Topic] (N pozīcijas)
[2-3 sentence factual summary of positions taken]
[3-5 topics total]

## Koalīcija vs Opozīcija
**Koalīcija:** [Factual summary]
**Opozīcija:** [Factual summary]
```

**Key differences from politracker briefs:**
- No "MMN perspektīva" section
- No "Ieteikumi kampaņai" section
- No attack angles or strategic recommendations
- Equal treatment of all parties
- Factual tone throughout

### 7.3 Storage

Briefs stored as `context_notes` with `note_type='daily_brief'` or `'weekly_brief'`. Also written as markdown to `wiki/dailies/` for Obsidian access. `generate.py` renders them as individual blog post pages.

## 8. Daily Routine

Operated by user + Claude Code interactively. No automation for launch.

### Steps:

1. **Ingest:** `ingest_all()` + `fetch_all_twitter()` — scrape news and X
2. **Claim extraction:** For each politician with new documents, extract positions neutrally. No campaign_voice loaded. Salience reflects general political importance.
3. **Contradiction detection:** Compare new claims against full history. Store via `store_contradiction()`. Mandatory.
4. **Tension tracking:** Log inter-politician dynamics with source URLs.
5. **Context notes:** Neutral observations about trends, events. No campaign framing.
6. **Daily brief:** Write neutral summary using `src/briefs.py` format. Stored as context_note + wiki daily.
7. **Wiki updates:** Update person/topic pages with session observations. `wiki_sync()` to refresh frontmatter stats.
8. **Generate site:** `python -m src.generate` — rebuilds all static HTML.

### Claude Code Session Start:
Read `wiki/index.md` for context. Drill into specific person/topic pages as needed.
